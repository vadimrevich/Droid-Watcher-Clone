Описание настройки Android Studio для компиляции приложения
===========================================================

1. Запустите Android Studio после установки или обновления.
2. Добейтесь, чтобы IDE запускалась без ошибок. Ошибки в-основном возникают в плагинах Kotlin, Groove и Java Core.
3. Для начала исправьте версию Java. Помните, что Android Studio v.3.3 корректно работает с JDK 1,6+, 1.7+ и 1.8+ и не работает с JDK10 (1.9+, 1.10+)!
4. Если переустановка JDK или JRE не помогла, удалите Android Studio (в chocolatey с опцией purge) и вновь устновите пакет "с чистого листа".
5. После того, как IDE запускается без ошибок, обновите сначала Android SDK, затем gradle.
6. Последовательно зайдите в настройки Android Studio (File -> Settings… и File -> Project Structure…) и выставите параметры, как показано на скриншотах. Особое внимание надо уделить путям и версиям репозиториев. 
7. Если у Вас другая версия Android Studio, уточните зависимости на его сайте. Единственно, не надо менять имена репозиториев maven на вкладке "Project Structure…" -> "Project", поскольку там представлен максимальный набор репозиториев. В крайнем случае оставьте только репозитории google и jcenter, если остальные репозитории выдают неправильные зависимости.
8. Будьте внимательны с последним экраном, dependences! Постарайтесь не менять в них ни версии библиотек, ни версии файлов. В противном случае компиляция программ не гарантируется. Автор понимает, что библиотеки старые, но пока у него не хватает квалификации изменить код пакета под новые библиотеки.
9. Скомпилированные apk файл будет работоспособен на версиях Android, начиная с 4.0 и заканчивая самыми современными версиями.
